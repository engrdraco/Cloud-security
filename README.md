## Cross-Cloud Core Competencies

While each CSP has its own nomenclature, the underlying architecture remains consistent. I have developed strong, cross-cloud proficiency in the following core pillars:

### 1. Identity & Access Management (IAM)
* **Concept:** Designing strict least-privilege access controls, role-based access control (RBAC), and condition-based policies.
* **Services Handled:** AWS IAM (Policies, Roles), GCP IAM (Service Accounts, Custom Roles), Azure Active Directory / Entra ID.

### 2. Secure Networking & Perimeter Defense
* **Concept:** Isolating workloads using micro-segmentation, private VPCs/VNets, and strict firewall rules to minimize attack surfaces.
* **Services Handled:** AWS VPC (Security Groups, NACLs), GCP VPC (Cloud Armor, Hierarchical Firewalls), Azure VNet (NSGs, Azure Firewall).

### 3. Infrastructure as Code (IaC) & Automation
* **Concept:** Eliminating configuration drift and ensuring repeatable, secure baselines by automating infrastructure deployment.
* **Tools Used:** **Terraform** (declaratively provisioning VPCs, networks, and logging buckets across AWS, GCP, and Azure).

### 4. Security Monitoring, Logging, & Triage
* **Concept:** Centralizing log aggregation, establishing continuous compliance baselines, and triaging real-time security alerts.
* **Services Handled:** AWS CloudWatch/CloudTrail, GCP Security Command Center & Cloud Logging, Azure Monitor & Microsoft Defender for Cloud.
